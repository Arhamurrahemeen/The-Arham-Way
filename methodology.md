# Core Methodology: Blueprint -> UI -> Code

### Phase 1: Idea & Blueprint (The Claude Engine)
[span_8](start_span)We use Claude specifically for its exceptional research and flow pattern outputs[span_8](end_span). 
* **[span_9](start_span)Output:** Comprehensive documentation of scope, flow, tech stack, and features[span_9](end_span).
* **[span_10](start_span)Efficiency:** By getting the blueprint right in Claude, we can migrate the idea to other AI tools without wasting tokens or hours on re-explaining the project[span_10](end_span).

### Phase 2: Visualizing the Spec (The Stitch Layer)
[span_11](start_span)Once the blueprint is solidified, we move to **Google Stitch**[span_11](end_span).
* **[span_12](start_span)Process:** Feed the Claude-generated specs into Stitch to generate high-quality frontend components[span_12](end_span).
* **[span_13](start_span)Focus:** Bridging the gap between a written technical requirement and a functional, aesthetic UI[span_13](end_span).

### Phase 3: The HSK Evolution
[span_14](start_span)This methodology was refined during the development and UI upgrade of the **HSK Bone Care** project, transitioning from a standard MERN stack approach to an AI-accelerated workflow[span_14](end_span).
